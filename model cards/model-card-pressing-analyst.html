Model Card for Pressing Analyst Wordalisation

The Pressing Analyst is implemented within the TwelveGPT Education framework and is intended as an illustration of data-driven conversational AI methods. Here we detail the specific application to football team pressing analysis. It is thus intended as an example to help others building wordalisations for team performance analytics.

Intended Use

The primary use case of this wordalisation is educational. It demonstrates how to convert team pressing statistics into natural language descriptions and enable conversational analysis. A secondary use case might be for users to understand team defensive pressing strategies in football analytics.

Out of scope: Professional football scouting, real-time tactical analysis, or use with current season data. The chat functionality is limited to pressing-related queries only.

Factors

The Pressing Analyst focuses on professional football teams in the Premier League. This version analyzes team-level pressing metrics rather than individual player performance. The demographic scope is limited to male professional athletes in elite European football. Team diversity reflects the global nature of the Premier League, with players and coaches from various nationalities and backgrounds.

Datasets

The analysis uses two primary data sources:

  1. Summary Metrics: Pre-calculated league z-scores for pressing performance (data/pressing/pressing_summary_metrics_new.csv)
  2. Event Data: Detailed match events from Wyscout/StatsBomb format (dataset/dynamic_events_pl_24/)

Data preprocessing involves normalizing coordinates, calculating per-match averages, and computing statistical transformations.

Model

Quantitative Model

The model processes team pressing data through several stages:

  1. Data Loading: Reads CSV files and parquet event data
  2. Statistical Processing: Calculates z-scores, ranks, and percentile ranks
  3. Coordinate Normalization: Standardizes pitch coordinates for consistent analysis
  4. Metric Aggregation: Computes per-match averages from season totals

Normative Model

The system applies football-specific norms where higher values indicate better performance for most metrics, except "negative metrics" like PPDA (lower is better). The LLM role is defined as a "UK-based football analyst" with expertise in pressing strategies.

def describe_level(value): thresholds = [1.5, 1, 0.5, -0.5, -1] words = ["outstanding", "excellent", "good", "average", "below average", "poor"] return describe(thresholds, words, value)

Key Calculation Scripts

  • build_pressing_metrics.py: Computes all detailed metrics from raw event data
  • pressing_analyst.py: Displays metrics and handles user interaction
  • pressing_pitch.py: Generates pitch map visualizations

Evaluation

The model's performance is evaluated through:

  • Qualitative Assessment: Manual review of generated descriptions
  • Offline Analysis: BLEU scores comparing LLM outputs to reference descriptions
  • User Testing: Chat interaction quality and factual accuracy

Evaluation scripts are located in the evaluation/ directory and use ground truth datasets for comparison.

Ethical Considerations

Data Privacy: Uses publicly available football statistics with no personal player data beyond team affiliations.

Fair Representation: Metrics are calculated consistently across all teams without bias toward specific playing styles.

Educational Use: Designed for learning purposes only, not for professional decision-making.

Transparency: All calculations and data sources are openly documented in the codebase.

Caveats and Recommendations

Important Limitations

  • Data is from historical seasons and may not reflect current team performance
  • Metrics depend on event data quality and classification accuracy
  • Pressing definitions may vary between data providers
  • Coordinate normalization assumes standard pitch dimensions

Recommendations for Use

  • Use for educational purposes and methodological understanding
  • Validate calculations against known team performances
  • Consider multiple metrics together rather than in isolation
  • Supplement with qualitative analysis for complete understanding

Future Improvements

  • Integration with real-time data sources
  • Player-level pressing analysis
  • Cross-season comparisons
  • Advanced machine learning for pattern recognition
